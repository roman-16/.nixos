# Replace WireGuard with ZeroTier + ZTNET

## Summary

Replace wg-easy (WireGuard) in the VPN microVM with ZeroTier + ZTNET for L2 VPN support, enabling LAN gaming with friends.

## Motivation

WireGuard is L3-only and cannot carry broadcast/multicast traffic needed for LAN game discovery. ZeroTier provides L2 virtual networking where all peers appear on the same Ethernet segment.

## Architecture

### What changes

**VPN microVM (`192.168.70.73`):**
- Remove: wg-easy container, DuckDNS, WireGuard kernel modules
- Add: PostgreSQL + ZeroTier + ZTNET Docker containers
- Ports: 9993/udp (ZeroTier), 3000/tcp (ZTNET web UI)
- Volume: 40GB (was 1GB)

**OpenClaw microVM:**
- Volume: 40GB (was 20GB) - standardize VM volume sizes

**Monitoring:**
- Gatus: Update health check from WireGuard to ZTNET
- Homepage: Update service entry

### Docker containers (all `--network=host`)

1. **PostgreSQL** (postgres:15.2-alpine) - ZTNET database
2. **ZeroTier** (zyclonite/zerotier:1.14.2) - ZeroTier controller + node
3. **ZTNET** (sinamics/ztnet:latest) - Web UI on port 3000

### Access

- ZTNET dashboard: `http://192.168.70.73:3000` (LAN only)
- Subpath via nginx not feasible (Next.js app requires basePath at build time)
- Subdomain (ztnet.halerc.xyz) possible as future enhancement via CF tunnel

## Decisions

- Keep microVM name as `vpn`
- Host networking for all containers (simpler, matches existing wg-easy pattern)
- LAN-only access for ZTNET dashboard
- 40GB volume size as standard for all microVMs
- Secrets (postgres password, nextauth secret) stored in secrets.json
